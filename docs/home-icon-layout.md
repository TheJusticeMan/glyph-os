# Home Icon Layout

This page summarizes what governs the home screen icon positions, what can change them, and how the layout pass uses those inputs.

## Positioning Inputs

| Input or rule              | Where it comes from                                                                    | What changes it                                                                                                                                                                    | How it affects position                                                                                                                                                                                                        |
| -------------------------- | -------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Canvas width and height    | `GestureCanvasView` reports size changes to `MainActivity.updateLauncherIcons()`       | Device size, orientation, system insets, app window resizing, or the first measured layout                                                                                         | Defines the coordinate space for every icon. The automatic formation is centered in this rectangle, spread across the available width and height, and finally clamped so icon centers stay at least one radius from each edge. |
| Visible home app set       | `MainActivity.selectHomeApps()`                                                        | Installed apps changing, launch counts changing, screen area changing, `HOME_ICON_MIN_VISIBLE`, `HOME_ICON_MAX_VISIBLE`, `HOME_ICON_AREA_BUDGET`, and `HOME_ICON_FOOTPRINT_FACTOR` | Only selected apps get layout nodes. Adding or removing selected apps changes the total count, which changes each automatic anchor in the golden-angle formation.                                                              |
| Stable app order           | `LauncherIconLayout.build()` sorts by package-name hash, then label, then package name | App install/uninstall, package name, or label changes                                                                                                                              | The sorted index chooses each app's automatic formation slot. Launch counts do not reorder the final layout, so frequent apps grow without jumping to a different slot.                                                        |
| Automatic formation anchor | `LauncherIconLayout.anchorFor()`                                                       | App index, selected app count, canvas size, and icon radius                                                                                                                        | Places icons in a deterministic golden-angle spiral/ellipse around the screen center. The first slot is the exact center; later slots use `sqrt(index / total)` for radial spacing and `index * GOLDEN_ANGLE` for angle.       |
| Icon radius                | `LauncherIconLayout.iconSize()` divided by two                                         | Launch count, min/max icon dp, and the fixed size cap used by layout                                                                                                               | Larger icons need more room. Radius reduces the usable formation area, increases collision distance, and increases the edge margin used by clamping.                                                                           |
| Previous nodes             | Current `canvasView.launcherIcons` passed back into `LauncherIconLayout.build()`       | Any earlier layout pass or animation state                                                                                                                                         | Existing icons start from their previous x/y before being pulled toward anchors. This keeps relayouts smooth instead of teleporting icons when counts, app sets, or screen size change.                                        |
| User anchor position       | `homeIconAnchors` in `MainActivity`                                                    | Long-press an icon into edit mode, drag it, and release                                                                                                                            | Replaces the automatic anchor for that package while the activity is alive. The icon is still allowed to move during collision solving, then it is gently pulled back toward the user anchor on each iteration.                |
| Fixed drag position        | `draggingHomeIconPositions` returned by `fixedHomeIconPositions()`                     | Active drag while the finger is down                                                                                                                                               | Temporarily pins the dragged icon to the finger-adjusted position during layout. Other non-fixed icons can be pushed away from it. When the drag ends, it becomes a normal user anchor instead of a fixed point.               |
| Collision separation       | `LauncherIconLayout.separate()`                                                        | Icon sizes, icon positions, selected app density, `PADDING_FACTOR`, and `REPULSION_DAMPING`                                                                                        | Repeatedly pushes overlapping icons apart. Minimum spacing is the sum of both radii plus padding based on the larger radius. Fixed icons do not move; non-fixed icons absorb the push.                                         |
| Anchor pull                | `LauncherIconLayout.pullToAnchors()`                                                   | Target anchor x/y and `ANCHOR_PULL`                                                                                                                                                | After separation, each non-fixed icon moves a small percentage back toward its anchor. This balances readable spacing with the intended formation or saved user position.                                                      |
| Iteration count            | `LauncherIconLayout.build(iterations)` defaults to `DEFAULT_ITERATIONS`                | Callers or tests can override it                                                                                                                                                   | More iterations give separation and anchor pull more chances to settle. Zero iterations returns the start positions after node creation.                                                                                       |
| Edge clamp                 | `LauncherIconLayout.clamp()`                                                           | Canvas size and icon radius                                                                                                                                                        | Keeps every icon center inside `[radius, width - radius]` and `[radius, height - radius]`, preventing icons from being laid out offscreen.                                                                                     |
| Animated display position  | `GestureCanvasView.advanceLauncherIconAnimation()`                                     | New target nodes from layout, `ICON_LAYOUT_ANIMATION_STEP`, and `ICON_LAYOUT_SETTLE_PX`                                                                                            | The layout returns target positions, then the view eases displayed x/y and size toward those targets. This changes what the user sees frame by frame, but not the target layout itself.                                        |

## Size, Visibility, and Drawing Inputs

| Input or rule                 | Where it comes from                                        | What changes it                                               | Effect                                                                                                                                                      |
| ----------------------------- | ---------------------------------------------------------- | ------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Launch count                  | `LaunchUsageStore` loaded into `MainActivity.launchCounts` | Launching an app through GlyphOS increments its package count | Grows icon size with a square-root curve. Because radius feeds the layout, frequent apps take more space and can push nearby icons farther away.            |
| Minimum and maximum icon size | `HOME_ICON_MIN_DP` and `HOME_ICON_MAX_DP`                  | Code constants                                                | Bound the computed target size before layout. These values also influence the home app selection budget.                                                    |
| Size count cap                | `DEFAULT_SIZE_COUNT_CAP` inside `LauncherIconLayout`       | Code constant                                                 | Layout sizes reach max at this count cap. Counts above the cap stay at max size.                                                                            |
| Home icon scale               | `settings.homeIconScale` and `GestureCanvasView.iconScale` | Pinch gesture on the home canvas                              | Scales drawn icons and hit testing from `0.0` to `1.0`. This changes apparent size, labels, and tap radius, but it does not recompute target x/y positions. |
| Label threshold               | `LABEL_MIN_ICON_DP` in `GestureCanvasView`                 | Code constant and current drawn size after scale              | Labels are drawn only when scaled icon size is large enough. Label drawing does not participate in collision spacing.                                       |
| Installed app cache           | `InstalledApps.getInstalledApps()`                         | Apps installed, removed, enabled, disabled, or refreshed      | Changes the candidate app list. This can change visible app selection and the total icon count, which then changes automatic anchors.                       |

## Layout Pass Order

1. `MainActivity.updateLauncherIcons()` waits until the canvas has a real width and height.
2. `selectHomeApps()` chooses which apps fit the home surface, prioritizing higher launch counts while preserving a minimum visible set.
3. `LauncherIconLayout.build()` computes each selected app's size, automatic anchor, user anchor, fixed drag state, and starting x/y.
4. Each iteration runs separation, anchor pull, and edge clamping.
5. `GestureCanvasView` receives the new target nodes and animates the displayed icons toward them.

## Reset Layout Flow

The management screen's Reset layout action refills the golden-angle spiral one slot at a time:

1. Build the automatic spiral slots for the current home app set.
2. Start at slot `0`, the center of the screen.
3. Pick the highest-launch-count visible app that has not already been assigned to a slot. If launch counts tie, pick the closest currently displayed icon.
4. Add that app's icon to the fixed-position map at the slot and rerun layout, so the icon slides into place while untaken icons settle around the fixed ones.
5. Wait for `GestureCanvasView` to report that the animation has settled.
6. Repeat for the next slot until every visible home icon has been assigned.

When the sequence finishes, the fixed reset positions become the activity's user anchors. They are no longer fixed, but future layout passes still pull those apps toward the reset spiral slots.

## Practical Tuning Guide

| Goal                                                 | Knob to change                                                                                          | Expected result                                                                                                   |
| ---------------------------------------------------- | ------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------- |
| Show more or fewer home icons                        | `HOME_ICON_MIN_VISIBLE`, `HOME_ICON_MAX_VISIBLE`, `HOME_ICON_AREA_BUDGET`, `HOME_ICON_FOOTPRINT_FACTOR` | Changes how many apps enter the layout before positioning starts. More icons increase density and collision work. |
| Make frequent apps grow faster or slower             | `LauncherIconLayout.iconSize()` curve or the size count cap                                             | Changes radius, which changes both visual weight and spacing pressure.                                            |
| Give icons more breathing room                       | `PADDING_FACTOR`                                                                                        | Raises the minimum separation distance between neighboring icons.                                                 |
| Make overlaps resolve more aggressively              | `REPULSION_DAMPING` or `DEFAULT_ITERATIONS`                                                             | Moves overlapping icons apart faster or gives the solver more passes to settle.                                   |
| Keep icons closer to their formation or user anchors | `ANCHOR_PULL`                                                                                           | Higher values pull icons back toward anchors more strongly after collisions.                                      |
| Change the automatic pattern                         | `anchorFor()`                                                                                           | Alters the baseline formation before drag anchors and collision solving are applied.                              |
| Change visual easing only                            | `ICON_LAYOUT_ANIMATION_STEP` and `ICON_LAYOUT_SETTLE_PX`                                                | Changes how quickly displayed icons catch up to target layout positions without changing target positions.        |
